# Настройка MinIO для загрузки изображений

MinIO - это S3-совместимое хранилище, которое запускается локально в Docker контейнере.

## Автоматическая настройка

MinIO автоматически запускается при использовании `docker-compose up`. 

## Доступ к MinIO Console

После запуска контейнеров:

1. Откройте браузер и перейдите на: **http://localhost:9001**
2. Войдите с учетными данными:
   - **Username**: `minioadmin`
   - **Password**: `minioadmin`

## Создание Bucket вручную (если нужно)

1. В MinIO Console нажмите **"Create Bucket"**
2. Имя bucket: `bazaar-images`
3. Настройте публичный доступ:
   - Откройте bucket
   - Перейдите в **"Access Policy"**
   - Выберите **"Public"** или настройте кастомную политику

## Проверка работы

После запуска всех контейнеров:

1. Bucket `bazaar-images` должен быть создан автоматически
2. Попробуйте загрузить изображение через форму создания объявления
3. Изображения будут доступны по адресу: `http://localhost:9000/bazaar-images/items/...`

## Изменение учетных данных

Для продакшена измените в `docker-compose.yml`:

```yaml
minio:
  environment:
    MINIO_ROOT_USER: ваш-username
    MINIO_ROOT_PASSWORD: ваш-пароль
```

И обновите соответствующие переменные в `backend` сервисе.

## Проблемы?

Если bucket не создается автоматически:
1. Зайдите в MinIO Console (http://localhost:9001)
2. Создайте bucket `bazaar-images` вручную
3. Настройте публичный доступ
